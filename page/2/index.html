TensorFlow Variable

发表于 2019-03-19 分类于 学习

tf.Variable(0, name="counter")表示定义了一个变量，变量的初始值是0，名字是counter
tf.constant(1)表示定义了一个常量，初始值为1，当然也可以初始化一个矩阵，几维矩阵都可以
tf.add(x, y) 表示将两个量相加
tf.assign(x, y)表示将y的值赋给x
tf.global_variables_initializer() 表示初始化上面涉及到的全部变量，常量之类的。
tf.run() 表示运行某一个tf表达式。

使用session运行的方法有两种
第一种是通过tf.Session()来获取session对象，然后使用 .run()方法。不过这种方法需要去close掉session（不手动close也行，程序结束自动关闭）。

第二种是通过 with tf.Session() as sess: 这种方法会在下面的程序代码中（可以理解成程序块）包含sess，同时也会在程序块结束时关闭session，当然也可以处理异常。

新建一个React 项目

发表于 2019-01-28 更新于 2019-01-29

一、 新建项目

my-app 是项目名称，可以按照自己需求更改

二、 项目入口
就像写代码有个main函数一样，/src/index.js则是整个项目的入口。

React 和 ReactDOM 是React的核心依赖。
App 则是我们自己编写的组件（组件一般用大写字母开头）。 这个组件挂载在/public/index.html中ID为root的DOM元素中。

三、 自定义组件
App.js

app.css

  1. 自定义组件需要继承React中的Component组件。
  2. 组件中要有render函数。
  3. 组件中可以没有constructor构造函数。

三、 render函数

  1. 函数的返回值就是该组件在页面中将会被渲染出来的内容。
  2. 返回值遵循 jsx 语法，html代码直接写，{}大括号中可以写 js 表达式（非js代码）。
  3. render 只能返回一个大的dom节点，即最外层不能有多个dom节点。如果不希望显示最外层那个dom节点可以在最外层用<React.Fragment>包裹。

四、constructor函数

  1. 就像java中的构造函数一样，在dom生成时执行。
  2. 重写的构造函数要先执行 super(props);。
  3. props 为父组件传递给该组件的参数的集合。
  4. state 为组件自身的变量的集合。
  5. props 自读不能更改。
  6. state 可以更改，但是要通过 this.setState() 函数来修改。

五、 自定义函数

  1. foo 为自定义函数，可以直接写在class中。
  2. 当需要调用时注意，在JSX中事件绑定不再和html写法中一样，列入onclick 和 onClick

六、 关于样式

  1. 可以使用行样式，即直接在dom节点上加上 style 属性。但是注意这个 style后面的大括号表示这个大括号里面需要写一个js表达式。而真正包含样式内容的则是这个大括号里面的json对象。
  2. 也可以使用 class 来给dom节点增加属性。但是由于es6中class 是 类 的关键词。所以此处我们用className来代替。当然这样的话需要我们单独来写一个css 文件，和然后用 import './app.css' 来导入到我们当前的组件中。

MySQL 安装以及开放远程链接

发表于 2019-01-20 更新于 2019-01-21 分类于 系统维护

安装环境

Ubuntu 16.04

安装

安装过程中会要你输入root账号的密码，两次，输入就好了。
安装完之后，输入 mysql -uroot -p 进入mysql管理。
输入 exit 回车就可以退出

更改配置

MySQL安装好了虽然可以本地访问管理，但是不能从别的机器访问管理。

原因

  1. MySQL 默认只允许本地回环地址访问，也就是 localhost，127.0.0.1 访问。
  2. 数据库里面root账号的默认host也是localhost

修改配置文件

  1. 修改 mysql 的配置文件。一般在/etc/mysql这个文件夹下的 my.cnf 或者 my.ini (就算不在这个文件里面，也可以在这个文件里面找到些许线索)。
  2. Ubuntu 16.04 的 MySQL配置文件是 /etc/mysql/mysql.conf.d/mysqld.cnf。
  3. 用vim 打开
  4. 在[mysqld]这个标签下面可以看到 bind-address = 127.0.0.1 用 # 给注释掉就好了
  5. 保存(vim不会编辑保存，去百度一下就会了)文件
  6. 重启mysql服务

修改登录账号

  1. 如果想用 root 账号远程登录，那么就是很简单的吧 mysql这个数据库里面的user表中 root 账号的host update成 % 就行了。进入mysql管理界面，然后输入: 

    update mysql.user set host='%' where user='root' and host='localhost';

    exit 退出,然后访问即可。

  2. 虽然上面的方法可行，但是安全性就没那么高。我常用的方法是，新建某一个数据库，再新建一个用户，赋予这个用户该数据库的权限即可。这样可以保证root账号的安全，毕竟root账号有整个数据库的权限。
